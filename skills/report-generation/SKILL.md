# Report Generation Skill

## Purpose

Turn data analysis outputs into a report that can be shown on GitHub or used in an interview.

## Input

- KPI table
- Analysis tables
- Charts
- Business findings
- Data quality risks

## Process

1. Write business background.
2. Summarize core KPIs.
3. Present 3-5 findings.
4. Add charts and tables.
5. Convert findings into actions.
6. Write README and resume description.

## Output

- `final_report.md`
- GitHub README outline
- Resume project description

## Quality Standard

The report must answer: what happened, why it matters, what should be done next.
