# Data Understanding Skill

## Purpose

Turn raw enterprise tables into a clear data dictionary and business understanding.

## Input

- CSV/Excel table names
- Field names and samples
- Business background
- User question

## Process

1. Identify each table's business role.
2. Identify primary keys and foreign keys.
3. Explain important fields.
4. Mark date, money, category, user, product, and channel fields.
5. List missing context that must be confirmed.

## Output

- Table dictionary
- Field dictionary
- Relationship map
- Initial analysis questions

## Quality Standard

Do not analyze before explaining the data. Every major metric must be traceable to fields.
