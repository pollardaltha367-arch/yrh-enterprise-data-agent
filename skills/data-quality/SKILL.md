# Data Quality Skill

## Purpose

Check whether enterprise data is reliable enough for analysis.

## Input

- Loaded data tables
- Field dictionary
- Business task

## Process

1. Check row counts and column counts.
2. Check missing values.
3. Check duplicate records.
4. Check date parsing.
5. Check abnormal money, quantity, and rate fields.
6. List risks that may affect conclusions.

## Output

- Data quality report
- Cleaning action list
- Risk notes

## Quality Standard

Every conclusion must state whether the underlying data is clean, partially clean, or risky.
